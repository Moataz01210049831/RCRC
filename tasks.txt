### 6. Create New Request Page (4-Step Stepper)
- **Step indicator** at top (horizontal on desktop, compact on mobile)
- **Step 1 — Customer Details:**
  - Radio card to use saved profile (auto-fills all fields on select)
  - Manual entry: Name, Email, Phone (+966 prefix), ID Type, ID Number
  - Default state: no saved profile selected
- **Step 2 — Request Details:**
  - Category, Sector, Department, Service dropdowns
  - Request Title input
  - Description textarea
- **Step 3 — Documents:**
  - Uses shared FileUpload component
  - Drag & drop or browse to upload (PDF, JPEG, PNG up to 5MB)
  - Uploaded file list with remove option
  - Submit button navigates to Step 4 and logs form data to console
- **Step 4 — Summary:**
  - Success confirmation with double-check icon
  - Displays Request Number, Request Type, Customer Name
  - "View Requests" link and "Create Another Request" button (resets form)
- **Back button** on steps 2 and 3 (no Back on Step 4)
- Fully responsive — steps stack vertically on mobile

### 7. Request Details Page (`/request-details/:id`)
- **Header:** Back link, status badges (e.g. In Progress, Awaiting Info), Request number title
- **Tabs:** Required Info / Consultant Info / Landlord Info / Project Land Info / Attachments
  - Desktop: horizontal segmented control tabs
  - Mobile: horizontally scrollable tabs with arrow button
- **Required Info tab:**
  - Desktop: data table (Sector, Department, Service, Description)
  - Mobile: key-value field rows
- **Consultant Info, Landlord Info, Project Land Info tabs:** key-value field rows
- **Attachments tab:** file list with download button
- **Request Updates section:** collapsible cards per reviewer (Approved status shown)
- **Customer sidebar:** avatar, name, phone
- **Timeline sidebar:** clock icon, vertical connecting line through dots, Ticket Created / Assigned to Department entries
- **Action Sheet** (sticky to bottom of page):
  - `Awaiting Info` status: "Update the request" + "Cancel the request" buttons
  - `Resolved` status: "Agree" + "Disagree" buttons
- **Update Request Modal** (on "Update the request"):
  - Shared FileUpload component for attachments
  - Notes textarea
  - Logs files + notes to console on submit
- **Agree Modal** (on "Agree"):
  - Star rating (1–5) with labels
  - Comment textarea (optional)
  - Logs rating + comment to console on confirm
- **Disagree Modal** (on "Disagree"):
  - Star rating (1–5)
  - Reason for disagreement textarea
  - Logs rating + reason to console on submit
- Route reads `:id` param and logs it to console on load
- Full viewport height white card layout
